class jina.logging.base.ColorFormatter(fmt=None, datefmt=None, style='%')[source]¶

Bases: logging.Formatter

Format the log into colored logs based on the log-level.

Initialize the formatter with specified format strings.

Initialize the formatter either with the specified format string, or a default as described above. Allow for specialized date formatting with the optional datefmt argument. If datefmt is omitted, you get an ISO8601-like (or RFC 3339-like) format.

Use a style parameter of ‘%’, ‘{‘ or ‘$’ to specify that you want to use one of %-formatting, str.format() ({}) formatting or string.Template formatting in your format string.

Changed in version 3.2: Added the style parameter.

format(record)[source]¶

Format the specified record as text.

The record’s attribute dictionary is used as the operand to a string formatting operation which yields the returned string. Before formatting the dictionary, a couple of preparatory steps are carried out. The message attribute of the record is computed using LogRecord.getMessage(). If the formatting string uses the time (as determined by a call to usesTime(), formatTime() is called to format the event time. If there is exception information, it is formatted using formatException() and appended to the message.

emit(record)[source]¶

Emit a record.

If a formatter is specified, it is used to format the record. The record is then written to the stream with a trailing newline. If exception information is present, it is formatted using traceback.print_exception and appended to the stream. If the stream has an ‘encoding’ attribute, it is used to determine how to do the output to the stream.

jina.logging.base.get_logger(context, context_len=15, log_profile=False, log_sse=False, log_remote=False, fmt_str=None, event_trigger=None, **kwargs)[source]¶

Get a logger with configurations

Parameters

  • context (str) – the name prefix of the log

  • context_len (int) – length of the context, i.e. module, function, line number

  • log_profile (bool) – is this logger for profiling, profile logger takes dict and output to json

  • log_sse (bool) – is this logger used for server-side event

  • log_remote (bool) – is this logger for remote logging

  • fmt_str (Optional[str]) – use customized logging format, otherwise respect the JINA_LOG_LONG environment variable

  • event_trigger – a threading.Event or multiprocessing.Event for event-based logger

Return type

Union[Logger, NTLogger]

Returns

the configured logger

Note

One can change the verbosity of jina logger via the environment variable JINA_LOG_VERBOSITY

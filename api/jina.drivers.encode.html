Note

batch_size is specially useful when the same EncoderExecutor can be used for documents of different granularities

(chunks, chunks of chunks …)

Warning

batch_size parameter was added to cover the case where root documents had very few chunks, and the encoder executor could then only process them in batches of the chunk size of each document, which did not lead to the full use of batching capabilities of the powerful Executors

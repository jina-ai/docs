class jina.proto.ndarray.generic.GenericNdArray(proto=None, is_sparse=False, dense_cls=<class 'jina.proto.ndarray.dense.numpy.DenseNdArray'>, sparse_cls=<class 'jina.proto.ndarray.sparse.scipy.SparseNdArray'>, *args, **kwargs)[source]¬∂

Bases: jina.proto.ndarray.BaseNdArray

A generic view of the Protobuf NdArray, unifying the view of DenseNdArray and SparseNdArray

This class should be used in nearly all the Jina context.

Simple usage:

# start from empty proto
a = GenericNdArray()

# start from an existig proto
a = GenericNdArray(doc.embedding)

# set value
a.value = np.random.random([10, 5])

# get value
print(a.value)

# set value to a TF sparse tensor
a.is_sparse = True
a.value = SparseTensor(...)
print(a.value)

Advanced usage:

GenericNdArray also takes a dense NdArray and a sparse NdArray constructor as arguments. You can consider them as the backend for dense and sparse NdArray. The combination is your choice, it could be:

# numpy (dense) + scipy (sparse)
from .dense.numpy import DenseNdArray
from .sparse.scipy import SparseNdArray
GenericNdArray(dense_cls=DenseNdArray, sparse_cls=SparseNdArray)

# numpy (dense) + pytorch (sparse)
from .dense.numpy import DenseNdArray
from .sparse.pytorch import SparseNdArray
GenericNdArray(dense_cls=DenseNdArray, sparse_cls=SparseNdArray)

# numpy (dense) + tensorflow (sparse)
from .dense.numpy import DenseNdArray
from .sparse.tensorflow import SparseNdArray
GenericNdArray(dense_cls=DenseNdArray, sparse_cls=SparseNdArray)

Once you set sparse_cls, it will only accept the data type in that particular type. That is, you can not use a GenericNdArray equipped with Tensorflow sparse to set/get Pytorch or Scipy sparse matrices.
